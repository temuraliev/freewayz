# Запуск на сервере (всё, чего нет на Vercel)

## Что где работает

| Компонент | Где запускается |
|-----------|------------------|
| **Next.js** (Mini App, API, страницы) | Vercel |
| **Cron** (проверка поставщиков Yupoo, и т.д.) | Vercel |
| **Админ-бот** (заказы, трекинг, поставщики, расходы, промокоды, импорт категории) | Сервер |
| **Клиентский бот** (/start, кнопки Каталог / Рекомендации / Профиль) | Сервер |
| **Импорт категории** (/importcategory в админ-боте) | Сервер (дочерний процесс бота) |

На сервере должны быть запущены оба бота. Тогда весь функционал будет работать: Vercel отдаёт приложение и API, сервер — боты и импорт по команде из бота.

---

## Требования на сервере

- **Node.js** 20+ (LTS)
- **Git** (чтобы клонировать репозиторий)
- Для **импорта категории** с бота на этой же машине нужны зависимости (Playwright и т.д.) — они ставятся через `npm install` в проекте

---

## 1. Подготовка проекта на сервере

```bash
cd /opt/freewayz   # или свой каталог
git clone https://github.com/temuraliev/freewayz.git .
npm install
```

Создать или скопировать **`.env.local`** в корень проекта и заполнить те же переменные, что и для локальной разработки / Vercel:

- `BOT_TOKEN` — токен клиентского бота (Mini App)
- `ADMIN_BOT_TOKEN` — токен админ-бота
- `ADMIN_TELEGRAM_IDS` — ID админов через запятую
- `NEXT_PUBLIC_SANITY_PROJECT_ID`, `NEXT_PUBLIC_SANITY_DATASET`, `SANITY_API_TOKEN`
- при необходимости: `GEMINI_API_KEY`, `TRACK17_API_KEY`, `ADMIN_WEBAPP_URL` / `NEXT_PUBLIC_APP_URL`

Оба бота при старте сами читают `.env.local` из корня проекта.

---

## 2. Запуск через PM2 (одной командой)

В корне проекта уже есть **`ecosystem.config.cjs`** — конфиг для двух ботов.

```bash
cd /opt/freewayz
npx pm2 start ecosystem.config.cjs
```

Проверка:

```bash
npx pm2 list
npx pm2 logs admin-bot
npx pm2 logs customer-bot
```

Автозапуск после перезагрузки сервера:

```bash
npx pm2 save
npx pm2 startup
# выполнить команду, которую выведет pm2 startup (с sudo, если нужно)
```

---

## 3. Запуск без PM2 (два терминала или screen/tmux)

```bash
# Терминал 1 — админ-бот
node scripts/admin-bot/index.mjs

# Терминал 2 — клиентский бот
node scripts/customer-bot/index.mjs
```

Оба скрипта сами подхватывают `.env.local` из корня проекта.

---

## 4. Полезные команды PM2

| Команда | Описание |
|---------|----------|
| `pm2 list` | Список процессов |
| `pm2 logs` | Логи всех приложений |
| `pm2 logs admin-bot` | Логи только админ-бота |
| `pm2 restart admin-bot` | Перезапуск админ-бота |
| `pm2 restart all` | Перезапуск обоих ботов |
| `pm2 stop all` | Остановить оба бота |

После обновления кода (`git pull`):

```bash
pm2 restart all
```

---

## 5. Импорт категории с бота

Команда `/importcategory` в админ-боте запускает на **этом же сервере** скрипт `scripts/import-yupoo-to-sanity.mjs`. Убедись, что на сервере выполнен `npm install` (в т.ч. Playwright). При первом запуске Playwright может доустановить браузеры — это нормально.

Итог: на Vercel — приложение и cron; на сервере — оба бота через `pm2 start ecosystem.config.cjs`, тогда весь функционал (включая импорт категории из бота) будет работать.
